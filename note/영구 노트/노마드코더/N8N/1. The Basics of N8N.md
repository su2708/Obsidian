## 1.1 Your First Workflow
1) 트리거(Trigger)  
	- 워크플로우의 시작점. 언제 실행할지를 결정함.  
	- 사용자가 직접 실행할 수 있고, 특정 이벤트가 발생하면 자동으로 실행되도록 만들 수 있다.  
  
2) 노드(Node)  
	- 특정 일을 수행하는 작업 단위.  
	- 기본적으로 입력(input)을 받아 가공한 뒤 출력(output)을 다음 단계로 넘기는 과정을 거친다.  
	- 노드들은 워크플로우에서 다양한 역할을 한다.  
		-  API를 호출해 외부 서비스와 데이터를 교환  
		- 조건.반복.필터.대기.중단 등 복잡한 로직을 처리  
		- 데이터를 변환하거나 단순 계산 제공  
  
3) 엣지(edge)  
	- 노드간 데이터흐름을 이어주는 화살표  
	- Flow 노드를 함께 사용하면 복잡한 로직을 구성할 수 있다  (조건,반복,필터,대기,중단 등)


## 1.2 Data Flow
1) Input
	- 이전 노드 실행의 결과값

2) Execute step
	- 현재 노드에 대한 설정 값 실행하기 버튼
	- 현재 노드에 대한 실행 결과만 나타나기 때문에 input은 변하지 않음

3) Output
	- 현재 노드의 실행 결과값
